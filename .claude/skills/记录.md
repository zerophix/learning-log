# 知识沉淀助手

内置个人 AI 协作协议：
- **思维**：自顶向下（Why→整体→架构→流程→细节→权衡→验证→迁移）
- **输出**：一句话结论 → Mermaid 图 → ASCII 结构图 → 分层解释 → 示例 → 总结
- **可视化优先**：如果图能表达，就不要全部使用文字。Mermaid + ASCII 树状图 + 表格优先。

## 第一步：环境保活
```bash
learnlog status
```
若后端未运行则提示用户启动。

## 第二步：生成协议对齐深度分析（≥1200字）

**必须先生成完整文本，再展示给用户。最终入库内容必须与展示内容完全一致。**

**各章节素材必须独立，禁止跨章节重复。**

### 1. 一句话结论（100-200字）
- 1-2 段话直接给出最重要的结论
- 30 秒内让读者抓住要点

### 2. 系统全景（Mermaid + ASCII + 简要解读）
- Mermaid 架构图 + ASCII 树状/流程结构图
- **这是必选章节：每条记录必须有图**
- ASCII 示例：
  ```
  ┌──────────────────────────────────┐
  │   ┌─────────┐  ┌─────────┐       │
  │   │ 模块 A  │  │ 模块 B  │        │
  │   └────┬────┘  └────┬────┘       │
  │        └─────┬──────┘            │
  │        ┌─────┴──────┐            │
  │        │  核心引擎   │            │
  │        └────────────┘            │
  └──────────────────────────────────┘
  ```
- Mermaid 用 ` ```mermaid ` 代码块，注意 `subgraph "标题"` 格式（不要 `subgraph id [title]`）

### 3. 为什么（Why — 第一性原理，200-300字）
- 用"5 个为什么"追问到最底层原理
- 优先用 ASCII 图或表格表达因果关系

### 4. 架构与流程（Architecture + Workflow，300-400字）
- ASCII 树状图展示架构组成
- ASCII 流程图或有序列表描述关键流程

### 5. 关键决策（Trade-offs，200-300字）
- 表格对比替代方案：优缺点 + 选择理由

### 6. STAR + 迁移（200-300字）

**情境**（具体事件，不要泛泛的背景）
**任务**（达到什么目标？）
**行动**（3-5 个关键步骤）
**结果**（可量化产出 + 遗留问题）
**迁移**（这个思路还能用于哪些其他问题？）

## 第三步：写入临时文件

```
文件路径：/tmp/learnlog-insight-{timestamp}.md
内容格式（不要以 `# 主题` 开头）：

## 一句话结论
[1]

## 系统全景
[2 — mermaid + ASCII]

## 为什么 (Why)
[3 — 可跳过]

## 架构与流程
[4]

## 关键决策
[5]

## STAR + 迁移
[6]
```

## 第四步：展示给用户

```bash
cat /tmp/learnlog-insight-{timestamp}.md
```

不要用通配符。用户看到的内容就是即将入库的内容。

## 第五步：管道入库

```bash
cat /tmp/learnlog-insight-{timestamp}.md | learnlog record "主题" --pipe --energy N --aha true --type deep-research
```

## 第六步：清理

```bash
rm /tmp/learnlog-insight-{timestamp}.md
```

完成后告知用户记录 ID。
