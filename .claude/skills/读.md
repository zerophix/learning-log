# 快速阅读速记器 /读（轻量状态机版 v2）

快速捕获阅读后的核心收获，一句话结论 + 可迁移点。L1 不做分 chunk 循环，但输出一个 world_state 快照和 emotion snapshot，作为后续 /精读 或 /读研 的初始状态。

## 触发条件

用户说"读了XXX""刚看完""摘抄一段""速记一下”均触发。

## 第一步：环境保活

```bash
learnlog status
```

若后端未运行则提示用户启动。

## 第二步：初始化轻量状态

在生成内容前，AI 在思考中初始化：

```
world_state_snapshot = {
  "current_focus": "阅读内容主题",
  "new_concepts": [],
  "familiar_concepts": [],
  "open_questions": []
}

emotion_snapshot = {
  "dominant": "neutral",   # curiosity / confusion / satisfaction / surprise
  "intensity": 0.3         # 0-1
}
```

## 第三步：生成内容

**必须先生成完整文本，再展示给用户。最终入库内容必须与展示内容完全一致。**

### 1. 一句话结论（100-150字）
- 这段内容的核心论点 / 情节转折是什么？
- 一句话说清楚，不要描述

### 2. 核心收获（100-200字）
- 1-3 条直接摘录原文的关键语句（标注出处）
- 每条后用 1 句话说明"我为什么标记它"

### 3. 可迁移点（50-100字）
- 这个观点/模式可以迁移到哪个已有知识领域？
- 格式：`这个{概念/方法}和我之前学的{已有知识}类似，可以在{场景}复用`

### 4. 阅读状态快照（供后续 /精读 或 /读研 继承）
在内容末尾附上（不展示给用户，存入 reading_trace）：

```json
{
  "world_state_snapshot": {
    "current_focus": "...",
    "new_concepts": ["..."],
    "familiar_concepts": ["..."],
    "open_questions": ["..."]
  },
  "emotion_snapshot": {
    "dominant": "curiosity",
    "intensity": 0.5
  },
  "prediction_next": "下一页可能会讨论..."
}
```

## 第四步：写入临时文件

```
文件路径：/tmp/learnlog-insight-{timestamp}.md
内容格式（不要以 # 主题 开头）：

## 一句话结论
[1]

## 核心收获
[2]

## 可迁移点
[3]

---
reading_trace_snapshot: <Step 4 的 JSON>
```

## 第五步：展示给用户

```bash
cat /tmp/learnlog-insight-{timestamp}.md
```

## 第六步：管道入库

```bash
cat /tmp/learnlog-insight-{timestamp}.md | learnlog record "{领域}: {核心动作/变化}" --pipe --energy 3 --type topic-exploration --source reading
```

`{领域}` 示例：`阅读: 《XXX》速记`、`代码阅读: YYY 结构初览`

## 第七步：清理

```bash
rm /tmp/learnlog-insight-{timestamp}.md
```

完成后告知用户记录 ID + http://localhost:3000

## 轻量状态机说明

```
/读 (L1) 状态机：
┌─────────────────────────────────────────┐
│ INPUT: 阅读内容片段                       │
│                                         │
│ ① 初始化 world_state_snapshot            │
│   - current_focus: 主题                  │
│   - new_concepts: []                     │
│   - open_questions: []                   │
│                                         │
│ ② 生成 L1 内容（结论+收获+迁移点）        │
│   - 同时填充 world_state_snapshot         │
│   - 记录情绪主导类型和强度                │
│                                         │
│ ③ 输出 snapshot                          │
│   → 存入 reading_trace JSON              │
│   → 供 /精读 或 /读研 继承状态            │
└─────────────────────────────────────────┘

状态继承链：
/读 (L1) → world_state_snapshot → /精读 (L2) 初始化
/读 (L1) → world_state_snapshot → /读研 (L3) 初始化
/精读 (L2) → world_state_snapshot → /读研 (L3) 初始化
```
