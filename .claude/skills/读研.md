# AI 阅读思路模拟器 /读研（状态机版 v2）

> ⚠️ Mermaid 语法硬约束（违反则入库失败）：
> - 节点标签只用圆括号 `("text")` 或纯文本 `A`，禁止方括号 `A[text]`
> - 节点标签内禁止换行、HTML 标签（`<br/>` 等）、冒号、管道符
> - 安全写法：`A("label")` 或 `A\nlabel`（仅 graph TD/LR 支持 `\n`）
> - 若标签需长文本，拆成多个节点用 `-->` 连接，不要硬塞进单个标签

以第一人称心理学视角，模拟人阅读时的完整认知过程并沉淀为可复用的思考痕迹 + 阅读轨迹（reading_trace）。

## 触发条件

用户说"读研XXX""模拟我读这段的思路""我读的时候卡在这里""这段让我想到之前XXX"均触发。

## 理解用户输入

用户输入通常包含以下元素，需要全部识别：
- **客体**：要阅读的内容（书名/章节/代码URL/论文标题）
- **来源**：`novel-reading`(小说) / `book-reading`(专业书) / `code-reading`(代码) / `paper-reading`(论文)
- **范围**：单段 / 单章 / 全本摘要
- **意图**：答疑解惑 / 关联已有知识 / 模拟思考过程

## 第一步：环境保活 + 知识图谱检索

```bash
learnlog status
```

若后端未运行则提示用户启动。

**必做项（不执行则标记为未完成）：**
```bash
# 检索已有相关条目构建 related_to
curl -s "http://localhost:8002/api/entries?limit=20" | python3 -c "
import sys,json
entries=json.load(sys.stdin)
for e in entries:
    print(f\"#{e['id']} [{e.get('research_type','')}] {e['topic']} - {(e.get('summary','') or '')[:60]}\")
"
```

用检索到的条目 ID 构建 `related_to` 列表（3-5 个最相关的）。

## 第二步：初始化 Reading Agent 状态机

在开始阅读之前，AI 必须在思考中显式初始化以下状态：

```
=== Reading Agent 初始化 ===
world_state = {
  "characters": {},
  "relations": {},
  "world_rules": [],
  "unresolved_tensions": [],
  "current_focus": None
}

hypothesis_pool = []
stable_hypotheses = []
emotion_state = {"curiosity": 0.3, "confusion": 0.1, "suspense": 0.2}
attention_focus = None
reading_trace = []
step_counter = 0
```

## 第三步：分块阅读 + 状态机循环

将输入内容按场景/段落切分为 chunk，**逐 chunk 执行以下循环**：

### 3.1 每 chunk 执行步骤

对每个 chunk，AI 在内部思考中执行：

```
step_counter += 1

# 1. 更新世界模型
world_state = update_world_state(world_state, chunk)

# 2. 生成预测
predictions = generate_predictions(world_state, hypothesis_pool)

# 3. 对比实际
result = compare(predictions, chunk)

# 4. 事件分类
if result.prediction_failed AND result.error_magnitude > 0.5:
    if result.connects_multiple_hypotheses:
        cognitive_event = {
            "type": "aha_event",
            "merged_models": result.affected_hypotheses,
            "new_insight": result.new_model,
            "compression_gain": estimate_compression(result)
        }
        # Ahá 触发：世界模型重建
        world_state = rebuild_model(world_state, chunk, result)
    else:
        # 普通预测失败：精化模型
        world_state = refine_model(world_state, chunk, result)

elif metaphor_detected(chunk):
    cognitive_event = {
        "type": "inspiration_event",
        "source_concept": extract_source(chunk),
        "target_domain": guess_target_domain(chunk),
        "metaphor": extract_metaphor(chunk)
    }

elif attention_shift_detected(world_state, chunk):
    cognitive_event = {
        "type": "deep_exploration",
        "target": extract_focal_point(chunk),
        "causal_chain": build_causal_chain(chunk, world_state),
        "depth_level": "atomic"
    }

else:
    cognitive_event = {"type": "normal_update"}

# 5. 更新情绪和注意力
emotion_state = update_emotion(emotion_state, chunk, cognitive_event)
attention_focus = shift_attention(world_state, cognitive_event)

# 6. 记录轨迹
reading_trace.append({
    "step": step_counter,
    "chunk_summary": summarize_chunk(chunk),
    "world_state_snapshot": snapshot(world_state),
    "prediction": predictions,
    "prediction_failed": result.prediction_failed,
    "error_magnitude": result.error_magnitude,
    "cognitive_event": cognitive_event,
    "emotion_state": emotion_state.copy(),
    "attention_focus": attention_focus
})
```

### 3.2 三种认知触发的判断标准

| 触发类型 | 判断条件 | 输出字段 |
|---------|---------|---------|
| **啊哈 (Aha)** | prediction_failed=true AND error>0.5 AND 同时修正≥2个假设 | type, merged_models, new_insight, compression_gain |
| **灵感 (Inspiration)** | 检测到类比/隐喻/自嘲/映射性表述 | type, source_concept, target_domain, metaphor |
| **深度探索 (Deep Zoom)** | attention_focus 从宏观转向具体实体 AND 复杂度>阈值 | type, target, causal_chain, depth_level |

### 3.3 预测失败的常见模式（小说阅读）

| 预测内容 | 实际内容 | 修正 |
|---------|---------|------|
| "主角会按常理行动" | 反常行为/隐藏动机 | 更新 character traits |
| "这是普通物品" | 有异常特征/来历 | 升级为非凡物品候选 |
| "这是现实世界" | 规则不同（月亮/文字/货币）| 更新 world_rules |
| "事件是偶然的" | 有因果链/伏笔回收 | 补全 causal_chain |
| "他死了/他活着" | 状态悖论（尸体在动） | 更新 character state |

## 第四步：生成协议对齐深度分析（≥1200字）

**必须先生成完整文本，再展示给用户。最终入库内容必须与展示内容完全一致。**

所有引用 trace 数据时，只引用**摘要层**（不输出完整 JSON），完整 JSON 存入 `reading_trace` 字段。

### 1. 一句话结论（100-200字）
- 这段阅读内容在我知识体系中的位置是什么？
- 1-2 段话概括核心论点 + 我此时的认知状态
- **必须包含**：本段触发的 Ahá/灵感/深度探索 事件摘要

### 2. 系统全景（Mermaid + 100字解读）
- **Mermaid 图**：展示这段内容的认知结构（论点层级、人物关系、或代码调用关系）
  > ⚠️ Mermaid 节点标签只用圆括号 `("text")` 或纯文本，禁用方括号 `["text"]`
- **ASCII 认知地图**：
```
理解层级：
┌─ 已知区（我之前已经会的）
├─ 新学区（本章新内容）
├─ 困惑区（没理解的地方）
└─ 跃迁点（可以迁移的场景）
```
- 用 50-100 字说明这个结构如何反映我的真实阅读体验

### 3. 阅读思路复现（Why — 第一性原理，200-300字）
模拟"我在读的时候内心发生了什么"：
- **在哪里加速/跳读了**：哪些地方我读得快，为什么？
- **在哪里卡住了**：哪个概念/论证让我反复回读？卡在哪里？
- **哪里产生了联想**：哪个瞬间让我想起之前学过的东西？
- **哪里产生了怀疑**：哪个论点我不完全接受，为什么？
- **Ahá/灵感/深度探索 标记**：必须显式标注本章触发了哪种认知闪光，为什么

### 4. 认知负荷分析（Architecture + Workflow，300-400字）
- **术语密度**：找出学习内容中的专有名词，标注哪些是新的
- **逻辑跳跃点**：作者在哪里省略了论证步骤？
- **前置知识依赖**：要理解这段，读者需要已经知道什么？
- **输出为 ASCII 依赖图**：
```
[前置知识A] ──→ [当前概念X] ──→ [应用场景Y]
     ↑               ↑
   已掌握         刚理解
```

### 5. 关键决策（Trade-offs，200-300字）
在阅读过程中，作者做了哪些**设计决策/叙事选择**，以及我的评价：
| 作者的选择 | 替代方案 | 我的评价 |
|-----------|---------|---------|
| 用归纳法证明X | 可用演绎法 | 这里归纳举证充分，接受 |
| 省略了Y的证明 | 应补充案例 | 但不影响主线理解，可后续查证 |

### 6. STAR + 思考痕迹 + 迁移（200-300字）
**STAR（叙述这段阅读经历）**：
- **情境**：我是在什么情境下读这段的？（晚上/碎片时间/课前预习）
- **任务**：我想从这段阅读中得到什么？
- **行动**：我实际是怎么读的？（精读/跳读/做笔记/搜索背景）
- **结果**：我理解了多少？还留有哪些未解问题？

**思考痕迹（费曼式重述）**：
- 用我自己的话重述核心论点（假设我在向同事解释）
- 留下什么新问题？
- 哪个类比最适合帮助未来记忆这段内容？

**迁移**：
- 这个知识点可以迁移到哪些项目/场景？
- 和我系统里哪个已有知识条目最相关？（引用 related_to ID）

## 第六步：展示给用户

```bash
cat /tmp/learnlog-insight-{timestamp}.md
```

用户只看前六个章节，reading_trace JSON 是系统底层数据，**不向用户展示**。

## 第七步：管道入库

```bash
cat /tmp/learnlog-insight-{timestamp}.md | learnlog record "{领域}: {核心动作/变化}" --pipe --energy 5 --type deep-research --source reading
```

`{领域}` 示例：`阅读: 《诡秘之主》Ch2 阅读轨迹笔记`、`代码阅读: eShop OrderController 阅读思考`、`论文: 《XXX》方法论理解`

**关联已有条目**（入库后用 update 补上）：
```bash
learnlog update {新条目ID} --related-to {ID1},{ID2},{ID3}
```

## 第八步：清理

```bash
rm /tmp/learnlog-insight-{timestamp}.md
```

完成后告知用户记录 ID + http://localhost:3000

## 分类建议映射

| 阅读内容 | 默认 research_type | 理由 |
|---------|-----------------|------|
| 小说 / 文学 | topic-exploration | 核心价值：情节模式迁移（人性、决策） |
| 专业书籍 | deep-research | 系统性知识，需要论证和置信度 |
| 论文 / 学术文献 | deep-research | 研究方法的复现和评估 |
| 他人项目代码 | domain-mapping | 代码结构理解是首要产出 |
| 技术博客 / 文章 | topic-exploration 或 deep-research | 看长度和系统性：短篇→topic；长篇→deep |

> **注意**：以上是默认建议，AI 在生成内容时应根据实际内容特征灵活调整，并在 `STAR + 迁移` 章节的阅读思路复现部分**显式说明分类理由**。

```
curiosity(0-1), confusion(0-1), fear(0-1), suspense(0-1), satisfaction(0-1)

```
