# 阅读体系命令

Learning Log 提供三层阅读记录能力，覆盖"速读情报 → 精读摘录 → 深度学习模拟"全谱系：

| 命令 | 级别 | 能量 | 适用场景 | 产出 |
|------|------|------|---------|------|
| `/读` | L1 Quick | 3 | 读完一段、摘抄、速记 | 一句话结论 + 核心收获 + 可迁移点 |
| `/精读` | L2 Phase | 4 | 读完一章、结构化梳理 | 一句话结论 + 章节结构 + 关键摘录 + 我的疑问 + 可迁移模式 |
| `/读研` | L3 Deep | 5 | AI 模拟思考过程、深度内化 | 思路复现 + 认知负荷分析 + 关键决策 + STAR + 思考痕迹 + 迁移 |

## 选型决策树

```
读完一段 / 摘抄核心观点 → /读（L1，100-300字）
    ↓ 需要更深入？
读完一章 / 章节梳理 → /精读（L2，500-800字）
    ↓ 有疑问 / 想关联已有知识 / 模拟思路？
AI 模拟阅读思路 / 深度思考痕迹 → /读研（L3，1200字+）
```

## 各内容类型的默认选型

| 内容类型 | 推荐命令 | 默认 research_type |
|---------|---------|-----------------|
| 小说 / 文学 | /精读 | topic-exploration |
| 专业书籍 | /读研 | deep-research |
| 论文 | /读研 | deep-research |
| 他人代码 | /读研 或 /精读 | domain-mapping |
| 技术博客 | /读 | topic-exploration |
| 博客长文 | /精读 | topic-exploration / deep-research |

## 跨命令关联机制

- `related_to`：链接同本书/同系列/相关主题的其他条目
- `related_tag_ids`：自动提取关键词作为标签，跨条目聚合
- `source`：统一标记为 `reading`

## 例子

```
用户: /读 《原则》第1章
→ AI: 生成 L1 格式内容
→ 入库: source=reading, research_type=topic-exploration, energy=3

用户: /精读 《设计模式》第3章 单例模式
→ AI: 生成 L2 格式内容 + 章节结构 ASCII 图
→ 入库: source=reading, research_type=topic-exploration, energy=4
→ 关联: related_to=[上章条目ID]

用户: /读研 eShop OrderController.cs 这段代码
→ AI: 检索知识图谱 → 找到相关条目
→ 生成 L3 格式：复述代码职责 → 识别困惑（异常处理）→ 关联已有知识
→ 入库: source=code-reading, research_type=domain-mapping, energy=5, related_to=[...]
```

## 与现有研究类型的关系

阅读维度和研究维度是**知识来源的两个正交轴**，可同时标记：
- 读专业书写深度笔记 → `research_type=deep-research` + `source=book-reading`
- 读代码画架构 → `research_type=domain-mapping` + `source=code-reading`
