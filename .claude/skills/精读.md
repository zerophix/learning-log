# 深度阅读结构化摘录器 /精读（轻量状态机版 v2）

读完一章/一节后，用结构化的方式记录理解，并与同本书其他章节建立关联。L2 引入 2-3 个 hypothesis（假设），每章验证/修正至少 1 个，输出 world_state 快照供 /读研 继承。

## 触发条件

用户说"精读XXX第N章""读完一章，梳理一下""章节摘要”均触发。

## 第一步：环境保活 + 继承前序状态

```bash
learnlog status
```

若后端未运行则提示用户启动。

**状态继承**（如有前序 /读 或 /精读 条目）：
- 读取前序条目的 `reading_trace_snapshot` 或 `world_state_snapshot`
- 作为本次初始 world_state 的基线
- 无前序条目则初始化空状态

```
继承后初始化：
world_state = {
  "characters": {},
  "relations": {},
  "world_rules": [],
  "unresolved_tensions": [],
  "current_focus": None
  # ← 填充前序 snapshot 数据
}
```

## 第二步：初始化本章假设池

每章开始前，根据前序状态生成 2-3 个可验证假设：

```
hypothesis_pool = [
  {"id": 1, "hypothesis": "...", "confidence": 0.x, "status": "pending"},
  {"id": 2, "hypothesis": "...", "confidence": 0.x, "status": "pending"}
]
```

示例（小说）：
- H1: "主角会拒绝合作" (confidence: 0.6)
- H2: "这个物品是非凡物品" (confidence: 0.4)

示例（代码）：
- H1: "这个方法是入口调用链的核心" (confidence: 0.7)
- H2: "异常处理存在漏洞" (confidence: 0.3)

## 第三步：分节阅读 + 假设验证

将本章内容按节点/段落切分为 section，逐 section 执行：

```
for section in chapter_sections:
    # 1. 更新世界模型
    world_state = update_world_state(world_state, section)
    
    # 2. 检查假设
    for h in hypothesis_pool:
        if h["status"] == "pending":
            if evidence_found(h, section):
                if confirms(h, section):
                    h["confidence"] += 0.2
                    if h["confidence"] >= 0.8:
                        h["status"] = "confirmed"
                else:
                    h["confidence"] -= 0.3
                    if h["confidence"] <= 0.2:
                        h["status"] = "rejected"
    
    # 3. 检测认知闪光（轻量版）
    events = []
    if any(h["status"] == "confirmed" for h in hypothesis_pool):
        # 2个以上假设被同时确认 → 结构闭合信号
        events.append({"type": "aha_signal", "confirmed_hypotheses": [...]})
    
    if metaphor_or_analogy_detected(section):
        events.append({"type": "inspiration_signal", "metaphor": "..."})
    
    if attention_shift(world_state, section):
        events.append({"type": "deep_zoom_signal", "target": "..."})
```

## 第四步：生成协议对齐内容

**必须先生成完整文本，再展示给用户。最终入库内容必须与展示内容完全一致。**

### 1. 一句话结论（100-150字）
- 这一章/节的核心论点或情节转折是什么？
- **必须包含**：哪些假设被确认/否定？认知闪光类型？

### 2. 章节结构（200-300字）
- 用 ASCII 或有序列表展示章节的逻辑层次
- 标注作者使用的论证方法（归纳/演绎/类比/数据）
- 标注本章的假设验证情况（✅确认 / ❌否定 / 🔄待验证）

### 3. 关键摘录（200-300字）
- 2-4 条原文摘录（标注页码/位置）
- 每条后用 1 句话说明：这条为什么值得标记？
- 标注这条摘录对应的假设验证

### 4. 我的疑问（100-200字）
- 1-3 个阅读过程中产生的困惑
- 格式：`我困惑的是{XXX}，因为{已有知识和当前内容的冲突点}`
- 困惑点转化为新的待验证假设，加入 hypothesis_pool（status=open）

### 5. 可迁移模式（100-150字）
- 这个章节的核心方法可以迁移到哪个已有项目/场景？

### 6. 本章状态快照（系统用，不展示给用户）

在内容末尾附上：
```json
{
  "world_state_snapshot": {
    "characters": {...},
    "relations": {...},
    "world_rules": [...],
    "unresolved_tensions": [...],
    "current_focus": "..."
  },
  "hypothesis_pool": [
    {"id": 1, "hypothesis": "...", "confidence": 0.8, "status": "confirmed"}
  ],
  "new_open_questions": ["..."],
  "cognitive_events": [
    {"type": "aha_signal", "confirmed": [1, 2]}
  ],
  "emotion_snapshot": {"dominant": "curiosity", "intensity": 0.6}
}
```

## 第五步：关联同系列条目

```bash
        print(e['id'])
"
```

## 第七步：展示给用户

```bash

```

## 第八步：管道入库

```bash

```

**关联已有条目**（入库后）：

```bash

```

## 轻量状态机说明

```
/精读 (L2) 状态机：
┌─────────────────────────────────────────────────────────┐
│ ① 继承前序 /读 或 /精读 的 world_state_snapshot          │
│                                         │
│ ② 初始化 2-3 个 hypothesis              │
│   - 基于前序结论进行可验证预测            │
│   - status: pending/confirmed/rejected   │
│                                         │
│ ③ 逐节更新：                             │
│   world_state.update(section)            │
│   → 检查每个 hypothesis 的证据            │
│   → 更新 confidence + status             │
│   → 检测 aha_signal（2+假设同时确认）     │
│   → 检测 inspiration_signal（类比/隐喻）  │
│   → 检测 deep_zoom_signal（注意力转移）   │
│                                         │
│ ④ 输出 world_state_snapshot              │
│   → 存入 reading_trace JSON              │
│   → 供 /读研 完整状态机继承               │
└─────────────────────────────────────────────────────────┘

认知闪光检测（L2 轻量版）：
┌──────────────────┬────────────────────────────┬──────────────────┐
│ 类型              │ 触发条件                    │ 输出              │
├──────────────────┼────────────────────────────┼──────────────────┤
│ aha_signal        │ 2+ hypothesis 同时 confirmed│ confirmed_ids    │
│ inspiration_signal│ 检测到类比/隐喻/自嘲         │ metaphor 文本    │
│ deep_zoom_signal  │ attention_focus 转移 +       │ target + reason  │
│                   │ 复杂度超过当前认知负荷        │                  │
└──────────────────┴────────────────────────────┴──────────────────┘

hypothesis 状态流转：
pending → confirmed（confidence >= 0.8）
pending → rejected（confidence <= 0.2）
pending → open（新增问题，留给 /读研 处理）
```

## 状态继承示例

```
/读 《诡秘之主》Ch1 速记
  → world_state_snapshot: {current_focus: "穿越确认", new_concepts: ["赫密斯文"]}

/精读 《诡秘之主》Ch2 精读
  → 继承 /读 Ch1 的 snapshot 作为初始 world_state
  → 初始化 hypothesis:
      H1: "转运仪式是穿越原因" (confidence: 0.4)
      H2: "克莱恩是自杀" (confidence: 0.3)
  → 验证后: H1→confirmed(0.8), H2→confirmed(0.9)
  → 输出 world_state_snapshot → 供 /读研 Ch3 继承

/读研 《诡秘之主》Ch3 读研
  → 继承 /精读 Ch2 的完整 hypothesis_pool + world_state
  → 运行完整状态机（分 chunk 循环 + 轨迹日志）
```
